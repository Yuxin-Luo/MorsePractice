# Stage 6：i18n 中英双语

## 目标

所有 UI 文字中英双语，运行时一键切换，刷新保留。这是用户计划里**最重要的差异化亮点之一**——GitHub 上 0 个摩斯练习器做中文 UI。

## 关键文件

- `src/i18n/index.js:1-100` — `t()` / `setLocale()` / `getLocale()` / `toggleLocale()` / `applyTranslations()` / `initI18n()`
- `src/i18n/zh.js:1-50` — 中文词典（默认）
- `src/i18n/en.js:1-50` — 英文词典
- `src/main.js:1-18` — 启动时调 `initI18n()` + `applyTranslations()`，绑定语言切换按钮
- `src/ui/app.js:6, 138-145` — `import { t }` + 在判分反馈里用 `t('feedback.correct')` 等
- `index.html:1-60` — 所有可翻译文字加 `data-i18n` / `data-i18n-placeholder` 属性
- `styles/main.css:75-99` — 右上角语言切换按钮
- `tests/i18n.test.js:1-80` — 11 测试

## 设计决策

- **dot-path 字典**：`t('mode.letter')` → 嵌套查找。结构清晰，加新条目成本低
- **缺失 key 返回 `[missing.key]`**：调试时一眼看到哪个 key 没翻译；不抛异常
- **DOM 转换三种属性**：`data-i18n`（textContent）/ `data-i18n-placeholder` / `data-i18n-aria`
- **localStorage 持久化 locale**：`morse.v1.locale` key
- **自动检测浏览器语言**：`navigator.language` 以 `zh` 开头 → `zh`，否则 `en`
- **toggleLocale 一键切换**：右上角按钮文字永远是"切到另一种语言"——"中文模式"显示"EN"，反之亦然
- **不引入运行时依赖**：手写 ~50 行 runtime，无 i18next 之类重库
- **占位符插值**：`t('key', { name: 'foo' })` 支持 `{name}` 模板

## 已知限制

- **不支持多复数形式**（`{n} items` vs `{n} item`）；v1 量小不需要
- **不支持嵌套 key 的 fallback**（中文缺 → 英文 → key 名）；当前直接显示 `[key]`
- **没有提取工具**：新增翻译需要手动维护 `zh.js` / `en.js` 两份；项目小可接受
- **没有 RTL/阿拉伯文支持**；暂时不涉及
