## 阶段 2：Android APK（**只在阶段 1 完全通过后开始**）

### 整体思路
**优先 PWA 方案**（更简单 / 跨平台 / 不依赖 Bubblewrap 这类冷门工具），PWA 不行再退回 Bubblewrap 1.24.1。

### 2a. 调研 PWA 方案（推荐先做）

候选工具（按推荐顺序）：
1. **PWA Builder**（https://www.pwabuilder.com/）—— 微软做的 web 服务，一键打包成 APK / iOS / Windows 包
   - 在线界面：填 PWA URL（`https://morsepractice.pages.dev/`）→ 自动检测 → 选 Android → 选 package id / version / 签名方式 → 下载 APK
2. **pwabuilder-cli**（命令行版 PWA Builder）—— 本地跑，免 web 界面
   - `npm install -g pwabuilder-cli` 或 `npx pwabuilder-cli`
3. **Capacitor**（npm 包）—— 把 PWA 当本地 web 应用打包

**⚠️ 签名提示**：PWA Builder 默认用自己的 debug key 签 APK。如果项目 keystore（`mykEY/morse-practice.keystore`）还在，**优先用 PWA Builder 的 "Use my own signing key" 选项传 keystore 签名**——后续版本升级时保持签名一致。

不想折腾签名 → 用默认 debug key 也行（PWA Builder 会自动签名）。

**做法**：
- 用户用浏览器打开 https://www.pwabuilder.com/，输入 `https://morsepractice.pages.dev/`，按界面走完下载 APK
- 或本地装 `pwabuilder-cli` / Capacitor，按工具文档跑
- 产物 → `release/dist/morse-practice-0.2.0.apk`

### 2b. PWA 不行 → 退回 Bubblewrap 1.24.1

- `release/.android-project/` 已 init 过（在真 TTY 跑过）
- `manifest-checksum.txt` ↔ `twa-manifest.json` 的 SHA-1 当前已 match（`c7da6d...`）
- `app/build.gradle` 已被 sed patch 到正确值

**做法**：
```bash
# 1. 确认环境变量就位
source ~/.bashrc   # JAVA_HOME / ANDROID_HOME / PATH

# 2. 跑 build
bash release/scripts/build-android.sh

# 3. 看第一行报错
#    - 如果是已知问题 → 翻 release/dev_doc/2026-06-25-ci-debug-log.md 找答案
#    - 如果是新问题 → 立刻停下，不循环猜
```
